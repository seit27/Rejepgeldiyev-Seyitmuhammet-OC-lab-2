# Лабораторная работа №2

## Создание потоков (WinAPI)

## 📚 Цель работы

Изучить создание и управление потоками в Windows с использованием WinAPI.
Научиться использовать функции:

* `_beginthreadex`
* `CreateThread`
* `SuspendThread`
* `ResumeThread`
* `WaitForSingleObject`
* `Sleep`

---

## 🛠 Постановка задачи

Создать консольную программу, состоящую из трёх потоков:

1. **main** — главный поток
2. **worker** — вычисляет сумму квадратных корней элементов массива
3. **Count** — выводит числа Фибоначчи

Условия:

* Глобальные переменные не использовать
* Передача данных в поток через структуру
* Использовать ожидание завершения потока
* Реализовать создание потоков двумя способами

---

# 🧩 Описание структуры программы

## 1️⃣ Структура для передачи данных

Для передачи данных в поток worker используется структура:

* размер массива
* указатель на массив
* переменная для хранения результата

Это позволяет избежать использования глобальных переменных.

---

## 2️⃣ Поток worker

Создаётся с помощью `_beginthreadex`.

Поток:

* получает массив чисел
* вычисляет квадратный корень каждого элемента
* прибавляет результат к общей сумме
* после каждой операции делает паузу 200 мс
* по завершении возвращает результат в структуру

---

## 3️⃣ Поток Count

Создаётся с помощью `CreateThread`.

Поток:

* выводит числа Фибоначчи
* после каждого числа делает паузу 100 мс
* работает заданное количество итераций

---

## 4️⃣ Управление потоками

Потоки создаются в подвешенном состоянии (`CREATE_SUSPENDED`).

Для управления используются:

* `ResumeThread()` — запуск потока
* `SuspendThread()` — приостановка
* `WaitForSingleObject()` — ожидание завершения

---

# ⚙ Как работает программа

## 🔹 Шаг 1

Пользователь вводит:

* размер массива
* элементы массива
* время приостановки потока

---

## 🔹 Шаг 2

Создаётся поток **worker** в подвешенном состоянии.

---

## 🔹 Шаг 3

Главный поток:

* запускает worker
* через заданное время приостанавливает его

Поток worker вычисляет:

]

После каждого вычисления поток «спит» 200 мс.

---


Выводится результат работы worker.

---

# 🧠 Используемые функции WinAPI

### 🔹 CreateThread

Создаёт новый поток в процессе.

### 🔹 _beginthreadex

Создаёт поток с поддержкой стандартной библиотеки C/C++.

### 🔹 SuspendThread

Приостанавливает выполнение потока.

### 🔹 ResumeThread

Возобновляет выполнение потока.

### 🔹 WaitForSingleObject

Ожидает завершения потока.

### 🔹 Sleep

Приостанавливает выполнение потока на заданное время.

---

# 📌 Что такое дескриптор и идентификатор

* **Дескриптор (HANDLE)** — объект Windows для управления потоком.
* **Идентификатор (Thread ID)** — уникальный номер потока в системе.

---

# 🔎 Проверка без WaitForSingleObject

Если убрать функцию ожидания, главный поток может завершиться раньше, чем worker.
Это приведёт к некорректной работе программы.
